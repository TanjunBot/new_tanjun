# Security Policy

| Version       | Supported    |

| 0.1.x - 0.6.x | :x:          |
| 0.7.x         | :warning:¹   |
| 0.8.x         | ✅²         |
| 0.9.x         | ✅          |

> ¹Support for this version will be dropped on 04/01/2025.
²Support for this version will be dropped on 18/01/2025.

## Reporting a Vulnerability

We value the security of our Discord Bot and appreciate responsible disclosure of any vulnerabilities. To report a security issue, please follow these steps:

1. **Create an Issue**: Use the [Issues tab](https://github.com/TanjunBot/new-tanjun/issues/new?assignees=&labels=security%2Fvulnerability&projects=&template=security.yml) on our GitHub repository. Make sure to select the **Security Vulnerability Report** template.

2. **Describe the Vulnerability**: Provide a clear and detailed description of the security vulnerability. Include:
   - Steps to reproduce the vulnerability (if known).
   - Impact level (choose from the provided options based on the severity).
   - Any known mitigation or workaround for the vulnerability.
   - Version of the bot affected.
   - Any additional context or information that could help us understand the issue better.

3. **Contact Information**: Optionally, provide your contact information if you would like us to reach out for more details.

4. **Sensitive Issues**: For sensitive security vulnerabilities, consider contacting us through a private ticket on [our Discord server](https://discord.arion2000.xyz).

Once reported, we will investigate promptly and keep you informed of any progress or fixes. We appreciate your contribution to keeping our bot secure.
